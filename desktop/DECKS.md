# Deck profiles and editor shortcuts

Each physical Stream32 device can keep up to 16 named profiles. The device
nickname identifies the hardware; profile names identify independent sets of
pages and keys. Selecting a profile sends all of its pages and its selected
device page to connected firmware. Offline edits remain saved and sync on the
next connection.

Existing `decks.json` files are migrated automatically. A legacy profile is
wrapped as a profile named **Default**, while its previous `name` becomes the
device nickname. Profile IDs are generated by Stream32 and do not change when
the friendly profile name is renamed.

Deck export is a single-profile schema-4 file; schema-1 through schema-4 files
can be imported. Import adds and selects a named profile instead of
replacing the device's other profiles. Full backups include the complete
named-profile registry.

## Focused-app switching

Each device has exactly one default profile. Stream32 selects that fallback
when the focused app does not match another profile. A profile can hold one
bounded app identity per platform:

- Windows matches an executable basename or path suffix.
- macOS matches a bundle identifier, with process name as a fallback.
- Linux X11 matches `WM_CLASS`, with process name as a fallback.

Choose **Use focused app** after focusing another application to capture its
stable identity, or enter the identity manually. Manual macOS process names
use `process:App Name`; Linux values use `class:name` or `process:name`.
Matches are exact, case-normalized identities rather than scripts or regular
expressions. If more than one profile has the same match, the
lexicographically first stable profile ID wins.

Stream32 checks focus conservatively in the background and does not switch
while its own editor is focused. Wayland has no generic focused-window API, so
automatic switching is disabled there without affecting manual profile
selection. macOS permission errors and a missing X11 `xdotool` are shown in
the profile editor.

## Key editing

- Focus a configured deck key and press `Ctrl/Cmd+C`, `Ctrl/Cmd+X`, or
  `Ctrl/Cmd+V` to copy, cut, or paste. Text fields keep their normal shortcuts.
- Paste rewrites the key index for its destination and asks before replacing an
  occupied key.
- Drag a configured key onto another key to move it. Occupied destinations
  swap the two keys.
- While dragging, move over a page or profile tab to expose a destination on
  another page or profile.
- Copy, Cut, and Paste buttons provide the same operations without keyboard
  shortcuts.

Page actions are accepted only when their target exists in the destination
profile. Invalid cross-profile pastes and swaps are rejected without changing
either profile.

## Live key state

Live state is a RAM-only overlay on the saved key appearance. Compatible
firmware advertises `key-update`; older firmware continues to receive the base
deck and the editor reports that live state is unavailable. Live titles,
colors, state, and artwork never rewrite profile JSON or flash artwork.

- **Local toggle** flips only after its action or complete Multi Action
  succeeds. Its status is local to Stream32, not remote application truth.
- **Clock** updates the key title at minute boundaries in 12- or 24-hour format.
- **Focused app** shows a bounded application identity from the existing
  first-party focus watcher; it never exposes window titles.

Toggle on-artwork uses the same bounded image upload pipeline as base artwork,
but firmware keeps the rendered RGB565 bytes only in RAM/PSRAM. Reconnects and
base profile syncs reapply current overlays without changing the saved base.

## Multi Actions

A Multi Action runs up to 16 ordered action or delay steps. Action steps use
the same media, hotkey, Type Text, Mouse, website, launch, page, and plugin
controls as regular keys. Delays are capped at 30 seconds each and two minutes
total. Multi Actions cannot be nested.

Page steps switch locally through the active device session. A sequence stops
at its first failed step, when its device disconnects, or when another named
profile becomes active. Repeated presses of the same Multi Action key do not
overlap, while other keys remain available.

## Keyboard and mouse input

**Type Text** sends up to 512 Unicode characters to the focused application.
Newlines and tabs are supported; unsafe control characters are rejected.
**Mouse** supports single/double left, right, and middle clicks, bounded
absolute or relative movement, and bounded vertical/horizontal scrolling.

Windows uses a persistent, structured `SendInput` helper. macOS uses fixed
System Events/CoreGraphics programs and requires Stream32 in **System Settings
> Privacy & Security > Accessibility**. Linux input is X11-only and requires
`xdotool`; generic Wayland input is intentionally reported as unavailable.
The action picker queries these host capabilities and displays the reason when
an action cannot run.
